Globus Toolkit® 3.2.1 Release Notes

Globus Toolkit 3.2.1 is a point release of the Globus Toolkit containing updates to the 3.2.0 release.

This page only lists the fixes since 3.2.0; for release notes specific to 3.2.0 go to:
http://www-unix.globus.org/toolkit/releasenotes/3.2/index.html.

Use the existing documentation for 3.2.0, which starts at http://www-unix.globus.org/toolkit/docs/3.2/index.html.

Core

Fixed the following bug for globus_core:

  • Bug 1706: Vendorcc flavor fails on AIX (updated AIX vendorcc support)

Common

Fixed the following bugs for globus_common:

  • Bug 1684: lookups always failing on IRIX 6.5 WS Core
  • Bug 1680: anonymous authentication still try to load X509_USER_PROXY certificate
  • Bug 1777: The method "put" in ContextManager class should be synchronized

Security (GSI)

Fixed the following bugs:

  • Bug 1802: accept_sec_context doesn't set LIMITED_PROXY_FLAG for GSI_3_LIMITED_PROXY
  • Bug 1774: Authorization is in some cases not performaned before delegation
  • Bug 1727: openssl segfaults in smime called by ncsa-cert-request.pl
  • Bug 1708: globus_ftp_control: gss_init_sec_context failed on AIX 5.1
  • Bug 1685: bn_div_3_words undefined in IRIX compile
  • Bug 1673: Import of anonymous name creates wrong name
  • Bug 1669: grid-proxy-init -help is broken in 3.2rc2

GridFTP

Fixed the following bug in gridftp:

  • Bug 1711: gss_buffer_desc.length larger than int, causes radix_encode/decode to be bad

[ top ]

RFT

Added security to the RFT factory and fixed self authorization and delegation.

[ top ]

XIO

Fixed the following bug in globus_xio:

  • Bug 1778: memory leaks in the GRAM client library (fixed XIO handle leak)

[ top ]

GRAM

Fixed the following bugs in GRAM3 (the OGSI version of GRAM):

  • Bug 1786: GT3.0 update error
  • Bug 1699: managed-job-globusrun does not parse args correctly

Fixed the following bug in GRAM2 (the pre-OGSI version of GRAM):

  • Bug 1659: globus_gram_job_manager pkg metadata problem in 3.2rc2 (fixed the dependency mismatch for the JM data package)

[ top ]

Packaging

Fixed the following bug in packaging:

  • Bug 1696: handling pyGlobus dependencies in the default installation (added pyglobus threaded packages)

We also made some changes to the globus-makefile-header: it was not putting external libs anywhere into the makefile header. It now appends them to GLOBUS_PKG_LIBS.
