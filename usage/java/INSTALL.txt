Requirements
============

 - Java SDK 1.4.2 or higher
 - Apache Ant 1.6.2 or higher

Make sure that 'java -version' and 'ant -version' return the right version
information.

Instructions
============

1. Create source directory:

$ mkdir ~/usage
$ cd ~/usage

2. Checkout the source code of the usage module and the dependent modules
   (all in one line):

$ cvs -d:pserver:anonymous@cvs.globus.org:/home/globdev/CVS/globus-packages \
  co usage database/java wsrf/java wsrf/schema wsrf/build.xml

3. Set the 'GLOBUS_LOCATION' environment property to some directory where
   everything will be installed. For example:

$ export GLOBUS_LOCATION=~/usageInstall

4. Compile and deploy the modules:

A. First compile/deploy Java WS Core:

$ cd wsrf
$ ant deploy
$ cd ~/usage

B. Compile/deploy the database module:

$ cd database/java/common/source
$ ant deploy
$ cd ~/usage

C. Compile/deploy the usage module:

$ cd usage/java
$ ant deploy
$ cd ~/usage

After all the modules are deployed successfully there should be 'globus-usage-receiver' and 'globus-usage-babysitter' scripts under the $GLOBUS_LOCATION/bin directory.

Database configuration
======================

psql -f $G_L/etc/globus_usage_receiver_samples/create_guss_tables_postgre.sql

Configuration
=============

'globus-usage-receiver' program starts the usage data receiver. The receiver 
is configured using the $G_L/etc/globus_usage_receiver/receiver.properties
file. 

'globus-usage-babysitter' program connects to the usage data receiver to obtain
receiver statistics.
